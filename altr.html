01The problem

Everyone uses AI. Nobody shares how.

A company's best prompts, SOPs, and agent instructions live in one person's head or a stale Google Doc. The moment someone improves how the team handles a task, that improvement reaches no one.

The skills exist. They just cannot travel. Not between people, not between the five AI tools each person already juggles, and definitely not to the non-technical majority who never touch Git.

04The wedge

Land with technical teams. Expand to the org.

Whole-org is the vision and the TAM. You still enter through one sharp door, the way Slack, Notion, and Figma all did.

Technical teams already have the skills, already run multiple tools, and feel the pain daily. Start there. As the rest of the company adopts AI, the same library already spans them.

The one thing no platform vendor will build: sync between ecosystems. Anthropic syncs across Claude. OpenAI across theirs. Nobody carries your skills from Claude Code to Cursor to ChatGPT. That neutrality is the moat.

The defensible position

Cross-vendor, non-technical, versioned skill sync.

No verified product combines all three. Versioning is table-stakes and given away free. The paid surface is management: private libraries, roles, approvals, and sync. You cannot charge for skills. You charge for order.

05How it works
AI writes the skills

Nobody hand-writes YAML. Describe intent in plain language, Altr generates a spec-compliant skill. Select one, say "add an edge case," it improves. The two most-starred community skills are literally "a skill that writes skills." This is validated behavior.

Git under the hood, invisible

Every save is a commit. Non-technical users see "v4 · Maya · yesterday" with one-click restore. Engineers get a real cloneable remote. Same library, two doors.

Plugin / MCP for all agents

Altr ships as a meta-skill and MCP server inside the tools people already use. "Make me a skill for X" commits it and syncs it out. No web app required to start.

Sync fan-out

Claude Code reads a marketplace URL, zero client software. A small sync agent writes into Cursor and Codex on disk. Export helpers cover Claude.ai and ChatGPT. Pull-only, manifest-based, no lock-in.

Architecture

Web + APINext.js on Vercel. Server-layer authz, no RLS reliance.
DataSupabase Postgres, metadata only. Storage for assets. The DB is a rebuildable index over Git.
AuthAuth.js, sole system, Supabase adapter. Google OAuth + email magic links.
Git backendAltr-hosted by default, or connect your own GitHub / GitLab / Bitbucket. Never runs on Vercel.
ModelsOpenRouter gateway (Claude + Gemini + GPT). BYOK default, capped managed quota on paid tiers.
Sync agentGo, single static binary, menu-bar. MIT-licensed for trust and distribution.

The invariant: drop the entire database, rebuild from the Git backends, lose nothing but derived indexes. The repo is always the source of truth. That is what makes "no lock-in" a feature, not a promise.

08Risks & tripwires

The threat is platform absorption.

Anthropic plans the full skill lifecycle natively and already ships org-admin deployment. That is the single biggest risk, and it is worth watching monthly.

The defensible zone stays cross-vendor. Anthropic will not carry your skills to Cursor and ChatGPT. That is the line to hold.

  • Anthropic ships cross-tool sync → pivot weight to governance and non-Anthropic tools
  • Agent Skills moves under the Linux Foundation → become the best-managed host, not a competitor
  • OpenAI or Cursor ship team libraries → double down on cross-vendor neutrality
  • Low technical moat, format is just files → moat is speed, breadth, non-technical UX, network effect
  • Demand unproven → if the v0 gate misses, do not build v1. Re-scope or stop
